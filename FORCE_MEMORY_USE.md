## The fundamental insight

The biggest flaw in pre-Memory-Road Wave was simple · the agent **had** memory but **forgot to use it**. Not on purpose · just out of momentum. Multiple Waves slipped. One incident · May 26 · a cleanup agent TRIMMED the soft-hook bullet from MEMORY.md ("Read project_continuity_state.md on startup") and the kernel daemon kept writing packets for 16 days that NOBODY READ. Total dark.

The fix is to mechanically force injection so the agent literally cannot skip it. **Two layers of hard hook · enforced by Claude Code's harness · not by any Claude's discipline.**

## The 2-Layer Hard Hook (BOTH are required)

### Layer A · SessionStart hook (fires ONCE at session boot)

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "cat ~/.claude/projects/<project-id>/memory/project_continuity_state.md 2>/dev/null",
            "timeout": 5
          }
        ]
      }
    ]
  }
}
```

Fires BEFORE the first user prompt arrives. The cortex packet is in context from the very first turn · Claude can't claim "no context" on session boot.

### Layer B · UserPromptSubmit hooks (fire EVERY user prompt)

```json
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/.claude/hooks/inject_continuity_packet.sh",
            "timeout": 5
          },
          {
            "type": "command",
            "command": "bash ~/.claude/hooks/force_memory_use.sh",
            "timeout": 5
          }
        ]
      }
    ]
  }
}
```

Fires on EVERY user message · re-injects the latest packet (kernel updates packet every 90s · this catches the freshness) + adds a `<system-reminder>` telling the agent to USE memory.

---

## Why BOTH layers

| Layer | When it fires | What it gets you | Why alone isn't enough |
|---|---|---|---|
| SessionStart | Once at session boot | Packet in context from turn 0 | Packet goes stale by turn 5 (kernel updates every 90s) |
| UserPromptSubmit | Every user prompt | Always-fresh packet · `USE MEMORY` reminder | First turn has no packet (no user prompt yet) |
| **Both** | Boot + every turn | Packet always fresh · always in context · `USE MEMORY` reminder every turn | (this is the goal) |

Belt + suspenders. The harness guarantees memory regardless of which Claude lands.

## Install

### 1. Make sure the kernel is writing the packet

```bash
# Kernel must be active · packet must be fresh (< 5 min old)
stat ~/.claude/projects/<project-id>/memory/project_continuity_state.md
systemctl is-active continuity-kernel    # → active
```

### 2. Copy hooks to Claude Code hook dir

```bash
mkdir -p ~/.claude/hooks
cp hooks/inject_continuity_packet.sh ~/.claude/hooks/
cp hooks/force_memory_use.sh ~/.claude/hooks/
chmod +x ~/.claude/hooks/*.sh
```

### 3. Merge both hook blocks into settings.json

See `setup/settings.json.example` for the exact JSON. Merge BOTH `SessionStart` and `UserPromptSubmit` arrays into `~/.claude/settings.json`.

**Important ·** use a Python script or `jq` for the merge · don't manually edit JSON. Easy to break a comma. The skill ships with `setup/install_hooks.sh` that does this safely.

```bash
# Safe merge
bash setup/install_hooks.sh
```

### 4. Backup settings.json

```bash
cp ~/.claude/settings.json ~/.claude/settings.json.bak.$(date +%Y%m%d-%H%M%S)
```

The hard hook is precious · keep a backup before any future edit.

### 5. Cold-start verify

Quit Claude Code entirely. Reopen. Send a test prompt. The response should reference the cortex packet ("READ THIS on startup" content visible at top of context).

## Why hard hooks beat soft hooks

| Soft hook (MEMORY.md bullet) | Hard hook (settings.json) |
|---|---|
| Lives in a text file Claude reads | Lives in the harness Claude runs in |
| Can be trimmed by a cleanup agent | Can only be removed by deliberate JSON edit |
| Subject to MEMORY.md auto-load limit | Always fires regardless of MEMORY.md state |
| New Claude has to choose to read it | New Claude can't choose · harness injects it |
| **Failure mode: silent 16-day blind spot (May 26)** | **Failure mode: noticeable JSON parse error at boot** |

If hard-hook fails · Claude Code logs the error loudly · noticed in seconds. If soft-hook fails · system goes dark silently · noticed 16 days later. **Hard hooks fail LOUD · soft hooks fail SILENT.**
